## 🔍 The Problem We're Solving

Excelerate offers learners access to a wide range of valuable opportunities — courses,
internships, mentorships, scholarships, and competitions. But two critical problems
prevent learners from getting the most out of the platform:

### Problem 1 — Opportunity Overload

> *"There are so many options. I don't know where to start or what's right for me."*

New learners — especially students entering university for the first time — arrive on
Excelerate and face a large, undifferentiated list of opportunities. Without guidance,
they cannot tell:

- Which opportunities match **their specific interests and career goals**
- What **experience level** is required for each programme
- What the **right order** of steps is for their particular growth path
- How each opportunity connects to the **next** in a coherent journey

The result is confusion, poor choices, and early drop-off from the platform.

### Problem 2 — Silent Struggles During Learning

> *"I'm in a session right now and I don't understand what's being taught — but there's
> no easy way to say so."*

Once learners are inside a course, masterclass, internship, or competition, there is no
structured, low-friction way for them to:

- Signal in real time that they are **confused, lost, or need more explanation**
- Give the tutor **immediate, actionable feedback** without disrupting the session
- Help tutors understand **which topics are landing** and which are not

Tutors, in turn, have no live visibility into learner comprehension — they teach without
knowing how much is being absorbed.

---

## 💡 Our Solution

**Excelerate Pathfinder** transforms a large collection of learning and career
opportunities into a **clear, personalised growth journey** — and keeps learners and
tutors connected throughout with a **real-time feedback loop**.

### Feature 1 — Personalised Learning & Career Roadmap

From the moment a learner signs up, Pathfinder asks about their interests, skills,
and career goals, then builds a **structured, step-by-step roadmap** of the Excelerate
opportunities most relevant to them — in the right order, at the right level.

No more guesswork. Every learner gets a path built specifically for them.

### Feature 2 — PulseCheck: Real-Time Session Feedback

During any live learning session, learners can tap a simple **PulseCheck button** to
instantly signal how they are following the content. Tutors see the responses on their
dashboard in real time and can adjust their teaching on the spot — before confusion
compounds into disengagement.

## 💓 PulseCheck — Real-Time Feedback System

PulseCheck is the core innovation of Excelerate Pathfinder. It gives every learner
a **voice in the room** without disrupting the flow of a session, and gives every tutor
**live eyes on the room** without having to stop and ask "does everyone understand?"

### How It Works

During any active learning session, the learner sees four response buttons on their
screen. They tap the one that best describes how they are following the content
at that moment:

| Button | Signal | What It Tells the Tutor |
|---|---|---|
| 👍 **Got It** | I understand — keep going | Content is landing well; maintain pace |
| 😕 **Confused** | Something isn't clicking | Slow down; a concept needs revisiting |
| ⏩ **Too Fast** | You're moving too quickly | Reduce pace; learner is falling behind |
| ❓ **Need More Explanation** | I need this unpacked further | Go deeper on this topic before moving on |

### What the Tutor Sees

- A **live dashboard** showing PulseCheck responses as they come in — updated in real time
- A **colour-coded summary** showing the proportion of "Got It" vs. concern signals
- A **post-session analytics view** showing which moments in the session generated the most confusion

### Why This Matters

Most online and blended learning environments are one-way. The tutor speaks;
the learner absorbs — or doesn't. PulseCheck makes the feedback loop **immediate,
specific, and continuous**, turning passive learners into active participants and
turning tutors into responsive, data-informed instructors.

## 🛠 Tech Stack

| Layer | Technology | Purpose |
|---|---|---|
| **Frontend** | Flutter (Dart) | Cross-platform — one codebase for iOS and Android |
| **State Management** | Provider / Riverpod | Scalable, reactive state management |
| **Backend & Database** | Firebase Firestore | Realtime data sync — powers live PulseCheck updates |
| **Authentication** | Firebase Auth | Secure sign-in for learners, tutors, and admins |
| **Push Notifications** | Firebase Cloud Messaging (FCM) | Smart reminders for deadlines and sessions |
| **Hosting / Functions** | Firebase Cloud Functions | Backend logic for roadmap generation and analytics |
| **Version Control** | GitHub | Source control and team collaboration |

> **Why Firestore for PulseCheck?**
> Firestore's realtime listeners ensure that the moment a learner taps a PulseCheck
> button, the tutor's dashboard updates **instantly** — no refresh, no delay. This is
> the technical backbone of the entire feedback feature and why Firestore was chosen
> over a traditional REST API approach.

## 🚀 Getting Started

### Prerequisites

| Tool | Version | How to Get It |
|---|---|---|
| Flutter SDK | 3.x or later | [flutter.dev/docs/get-started/install](https://flutter.dev/docs/get-started/install) |
| Dart | 3.x (included with Flutter) | Installed automatically with Flutter |
| Android Studio or VS Code | Latest | Install Flutter + Dart plugins |
| Git | Any | [git-scm.com](https://git-scm.com) |
| Firebase Account | — | [console.firebase.google.com](https://console.firebase.google.com) |

### Installation Steps

```bash
# 1. Clone the repository
git clone https://github.com/Jumoke1/excelerate-mobile-team4.git
cd excelerate-mobile-team4

# 2. Install all Flutter dependencies
flutter pub get

# 3. Connect Firebase
#    → Go to console.firebase.google.com
#    → Create a project called "excelerate-pathfinder"
#    → Add an Android app and download google-services.json
#      Place it at: android/app/google-services.json
#    → Add an iOS app and download GoogleService-Info.plist
#      Place it at: ios/Runner/GoogleService-Info.plist

# 4. Verify your Flutter setup
flutter doctor

# 5. Run the app on a connected device or emulator
flutter run
```

### Running Tests

```bash
# All unit and widget tests
flutter test

# Integration tests (requires a connected device or running emulator)
flutter test integration_test/
```
